# Build a Meeting Notes App with Pipecat and AssemblyAI Universal-3 Pro

Build a real-time meeting transcription and notes app using Pipecat's pipeline framework, AssemblyAI Universal-3 Pro streaming STT, Cerebras LLM, and Rime TTS. The AI facilitator joins meetings, transcribes speech in real time, participates in conversation, and generates structured meeting notes when the session ends.

### ‚ù∂ Install dependencies

<codefocussection filepath="requirements.txt" filerange="1-5" title="Python dependencies" themeColor="#0000FF" label="Server">

Install Pipecat with plugins for each service: AssemblyAI for speech-to-text, Cerebras for the LLM, Rime for text-to-speech, Silero for voice activity detection, and WebRTC for browser-based audio transport. Also install `python-dotenv` for environment variables, `fastapi` and `uvicorn` for the HTTP server, and the `openai` client library for post-session LLM calls.

```bash
pip install "pipecat-ai[assemblyai,cerebras,rime,silero,daily,webrtc]" python-dotenv fastapi uvicorn openai
```

</codefocussection>

### ‚ù∑ Configure API keys

<codefocussection filepath=".env.example" filerange="1-3" title="Environment variables template" themeColor="#0000FF" label="Server">

Create a `.env` file with your API keys. You'll need keys for [AssemblyAI](https://www.assemblyai.com/) (speech-to-text), [Cerebras](https://cloud.cerebras.ai/) (LLM), and [Rime](https://rime.ai/) (text-to-speech).

</codefocussection>

### ‚ù∏ Import libraries and load environment

<codefocussection filepath="meeting_notes.py" filerange="1-33" title="Import libraries and load environment" themeColor="#0000FF" label="Server">

Import the Pipecat modules needed for the meeting notes pipeline: `TranscriptionFrame` and `InterimTranscriptionFrame` for transcript collection, `FrameProcessor` as the base class for custom processors, `OpenAILLMContext` for the conversational LLM pipeline, and `AsyncOpenAI` for calling Cerebras directly at session end. `load_dotenv()` pulls API keys from your `.env` file.

</codefocussection>

### ‚ùπ Define agent persona and meeting notes prompt

<codefocussection filepath="meeting_notes.py" filerange="35-72" title="Define agent persona and meeting notes prompt" themeColor="#0000FF" label="Server">

Configure transport parameters for Daily and WebRTC with audio I/O enabled and Silero VAD. Define `SYSTEM_INSTRUCTIONS` for the AI facilitator's personality ‚Äî it listens, asks clarifying questions, captures action items, and keeps responses concise for voice delivery. Define `MEETING_NOTES_PROMPT` to instruct the LLM to generate structured notes with five sections: Summary, Key Discussion Points, Decisions Made, Action Items, and Next Steps.

</codefocussection>

### ‚ù∫ Build transcript processors

<codefocussection filepath="meeting_notes.py" filerange="75-123" title="Build transcript processors" themeColor="#0000FF" label="Server">

Two custom Pipecat processors handle transcript collection. `TranscriptCollector` stores finalized user transcripts with timestamps for later note generation and broadcasts them to the browser via WebSocket, while also forwarding interim results so users see words appearing in real time. `AgentResponseCollector` buffers the AI facilitator's `TextFrame` chunks until the response is complete, then stores and broadcasts the full response as a transcript turn.

</codefocussection>

### ‚ùª Generate meeting notes with the LLM

<codefocussection filepath="meeting_notes.py" filerange="126-157" title="Generate meeting notes with the LLM" themeColor="#0000FF" label="Server">

Called once after the meeting ends rather than during the pipeline. Formats all collected transcript turns (both user and agent) with timestamps, then sends them to Cerebras (`llama3.1-8b`) via the OpenAI-compatible API with a lower temperature (0.3) for focused, deterministic output. Returns the structured notes for broadcasting to the client.

</codefocussection>

### ‚ùº Configure STT, LLM, TTS, and assemble the pipeline

<codefocussection filepath="meeting_notes.py" filerange="160-217" title="Configure services and assemble pipeline" themeColor="#0000FF" label="Server">

Set up AssemblyAI U3P with balanced turn detection ‚Äî `min_end_of_turn_silence_when_confident=560` and `max_turn_silence=2000` accommodate natural meeting speech patterns where speakers pause to think. Configure Rime TTS (`mistv2` model, `astra` voice) and Cerebras LLM (`llama3.1-8b`, temperature 0.7). Assemble the full 9-stage pipeline: `input ‚Üí STT ‚Üí transcript_collector ‚Üí context.user() ‚Üí LLM ‚Üí agent_response_collector ‚Üí TTS ‚Üí output ‚Üí context.assistant()`.

</codefocussection>

### ‚ùΩ Handle connection events and run

<codefocussection filepath="meeting_notes.py" filerange="219-248" title="Handle connection events and run" themeColor="#0000FF" label="Server">

On connect, the agent generates an initial greeting introducing itself as an AI meeting facilitator. On disconnect (when the user clicks "End Meeting"), `generate_meeting_notes()` is called with all collected transcript turns, and the structured notes are broadcast back to the browser via WebSocket. The `__main__` block uses the run helper from `run.py`, which sets up WebRTC transport, WebSocket endpoint, and a FastAPI server with the custom frontend.

</codefocussection>

### ‚ùæ Build the WebRTC connection with trickle ICE

<codefocussection filepath="client/app.js" filerange="38-160" title="WebRTC connection with trickle ICE" themeColor="#22C55E" label="Client">

Establish a WebRTC peer connection using the offer/answer exchange with the Pipecat backend (`POST /start` then `POST /sessions/{id}/api/offer`), sending only the audio track while keeping video local-only. The offer is sent immediately after `setLocalDescription()` without waiting for ICE gathering ‚Äî critical for production behind proxies where gathering may never complete. ICE candidates are trickled individually via `PATCH` requests, and a 30-second timeout resets the UI if the connection fails. A WebSocket to `/ws/transcripts` receives real-time transcript updates and generated notes.

</codefocussection>

### ‚ùø Handle messages and end meeting

<codefocussection filepath="client/app.js" filerange="210-314" title="Handle messages and end meeting" themeColor="#22C55E" label="Client">

The WebSocket message handler routes three types: `transcript` (finalized turns added to the scrolling panel), `partial` (interim results shown in the caption bar), and `notes` (generated markdown rendered with the `marked` library). When the user clicks "End Meeting", the WebRTC connection closes (triggering server-side note generation) while the WebSocket stays open to receive the generated notes.

</codefocussection>

### ‚ì´ Run the agent

```bash
python meeting_notes.py
```

Open [http://localhost:7860](http://localhost:7860), click **Start Meeting**, and begin your meeting. The AI facilitator will greet you and participate in conversation while transcribing everything in real time. Click **End Meeting** to generate structured notes.
