## Overview

In this guide, you'll build a listen-only meeting transcription agent that captures speech in real time, displays a live transcript in the browser, and generates structured meeting notes when the session ends. Unlike a conversational voice agent, this is a passive listener — no TTS, no audio output. The agent uses AssemblyAI's Universal-3 Pro model for streaming transcription with balanced turn detection tuned for meeting cadence, then sends the full transcript to an LLM for structured note generation.

### What you'll use

- **AssemblyAI Universal-3 Pro** — Streaming STT with punctuation-based turn detection
- **Pipecat** — Pipeline framework for building voice and audio agents
- **Cerebras** — LLM for generating structured meeting notes at session end

### How it differs from a voice agent

| | Voice Agent | Meeting Notes |
|---|---|---|
| Pipeline | Input → STT → LLM → TTS → Output | Input → STT → TranscriptCollector |
| Audio output | Yes (TTS) | No — listen-only |
| Frontend | Prebuilt Pipecat WebRTC UI | Custom UI with live transcript |
| Data to client | Audio (WebRTC) | Transcript + notes (WebSocket) |
| LLM usage | Real-time conversational responses | Post-session note generation |
| Turn detection | Aggressive (100ms / 1200ms) | Balanced (560ms / 2000ms) |

## Prerequisites

To follow along, you'll need:

- Python 3.10 or higher
- An [AssemblyAI API key](https://www.assemblyai.com/dashboard/signup)
- A [Cerebras API key](https://cloud.cerebras.ai/)

## Project setup

Create a new project directory and set up a virtual environment:

```bash
mkdir meeting-notes && cd meeting-notes
python -m venv .venv
source .venv/bin/activate
```

Install the dependencies:

```bash
pip install "pipecat-ai[assemblyai,cerebras,silero,daily,webrtc]" python-dotenv fastapi uvicorn openai
```

Note that unlike the voice agent, there's no `rime` (TTS) or `pipecat-ai-small-webrtc-prebuilt` dependencies — this agent uses a custom frontend instead of the prebuilt UI.

Create a `.env` file with your API keys:

```env
ASSEMBLYAI_API_KEY=your_assemblyai_api_key
CEREBRAS_API_KEY=your_cerebras_api_key
```

Now create a `meeting_notes.py` file and follow along below.

  The STT configuration uses **balanced turn detection** tuned for meeting cadence rather than the aggressive settings used in the voice agent:

  - **`keyterms_prompt`** — Boosts recognition of meeting-specific vocabulary: participant names, project names, and common meeting terminology.

## The custom frontend

The meeting notes agent uses a custom browser-based UI instead of Pipecat's prebuilt WebRTC widget. The frontend handles two connections:

1. **WebRTC** — Sends microphone audio to the Pipecat pipeline on the server
2. **WebSocket** — Receives live transcript updates and generated meeting notes from the server

## Run the agent

Start the meeting notes agent:

```bash
python meeting_notes.py
```

Open [http://localhost:7860](http://localhost:7860) in your browser, click **Start Meeting**, and begin your meeting. You'll see transcript turns appear in real time. Click **End Meeting** to generate structured notes — they'll appear in a formatted panel below the transcript.

## How turn detection works

AssemblyAI's Universal-3 Pro uses punctuation-based turn detection. For the meeting notes agent, we use balanced settings that accommodate meeting speech patterns:

1. You speak, and audio streams to AssemblyAI in real time
2. When you pause for 560ms (`min_end_of_turn_silence_when_confident`), the model checks if the transcript ends with terminal punctuation (`.` `?` `!`)
3. If terminal punctuation is found, the turn ends and the transcript is added to the buffer and displayed in the browser
4. If no terminal punctuation, a partial transcript is emitted and shown as interim text
5. If silence reaches 2000ms (`max_turn_silence`), the turn is forced to end regardless

The higher thresholds (560ms/2000ms vs 100ms/1200ms for the voice agent) mean the model is more patient with meeting speakers, producing cleaner and more complete transcript turns.

## Speaker diarization

Speaker diarization is not currently available in AssemblyAI's streaming API. For speaker-labeled meeting notes, use a hybrid approach: stream during the meeting for live transcription, then process the recording through the [async API](https://www.assemblyai.com/docs) for speaker-diarized, summarized notes.
